01 — Introduction

What Problem Do RNNs Solve?

Standard feedforward neural networks process every input independently. They have no memory of what came before. Show them the sentence "The cat sat on the ___" and they cannot leverage context from previous words to predict the next.

RNNs solve the sequential dependency problem by maintaining a hidden state — a compressed memory — that carries information across time steps. The same network weights are reused at every step, letting RNNs generalize across sequences of arbitrary length.

RNNs are the natural architecture for: text and language, audio and speech, time-series forecasting, DNA sequence analysis, video frame prediction, and any domain where order matters.

ℹ The core insight: a single RNN "cell" is applied repeatedly, sharing weights across every time step. This is analogous to how humans process language — we don't forget what was said three words ago.
02 — Architecture

The Unrolled RNN

An RNN can be visualized by "unrolling" it across time. Each column below represents one time step — the same cell is reused, sharing weights W_xh, W_hh, and W_hy throughout.

Key insight: the weights W_xh, W_hh, and W_hy are shared across all time steps. This is weight tying across time — it lets the network generalize to sequences of any length without adding parameters.

03 — Mathematics

The Core Equations

At each time step t, the RNN computes two things: an updated hidden state (memory) and an output.

Hidden State Update
h(t) = tanh( W_xh · x(t) + W_hh · h(t−1) + b_h )
↑ input contribution ↑ memory from past
Output
y(t) = W_hy · h(t) + b_y
↑ optionally followed by softmax for classification

The tanh activation squashes the hidden state to [−1, 1], preventing unbounded growth. Note that h(0) — the initial hidden state — is typically set to all zeros.

Bidirectional RNN

A bidirectional RNN runs two passes over the sequence — one forward (past → future) and one backward (future → past). The outputs are concatenated, giving each position access to the full sequence context. This is essential for tasks like named entity recognition or question answering where you need to see the whole sentence before deciding.

⚠ Bidirectional RNNs require the full input sequence at once — they cannot process streaming inputs in real time.
05 — The Problem

Vanishing Gradient Problem

During backpropagation through time (BPTT), gradients flow backwards through every time step — multiplied by W_hh at each step. If the spectral radius of W_hh is less than 1, the gradient shrinks exponentially:

Gradient signal vanishes as it travels back through many time steps → the network forgets long-range dependencies.

This is why vanilla RNNs practically fail on sequences longer than ~10–20 steps. The solution: LSTM and GRU, which introduce gating mechanisms that allow gradients to flow unimpeded over hundreds of steps.

06 — Long Short-Term Memory

LSTM Architecture

Proposed by Hochreiter & Schmidhuber (1997), the LSTM introduces a separate cell state C(t) — a memory highway that runs parallel to the hidden state. Three gates control what information flows in, out, and what gets erased.

f

Forget Gate

Decides what fraction of the old cell state to discard. Output is sigmoid — 0 means erase completely, 1 means preserve fully.

f(t) = σ( W_f · [h(t-1), x(t)] + b_f )
i

Input Gate

Decides which new information to write into the cell state. Works with a tanh layer that generates candidate values.

i(t) = σ( W_i · [h(t-1), x(t)] + b_i )
o

Output Gate

Decides what portion of the cell state to expose as the hidden state h(t). Filters the cell state through tanh.

o(t) = σ( W_o · [h(t-1), x(t)] + b_o )
Cell State Update (the key innovation)
C(t) = f(t) ⊙ C(t−1) + i(t) ⊙ C̃(t)
h(t) = o(t) ⊙ tanh( C(t) )
⊙ = element-wise multiplication

The cell state only interacts with the hidden state through additive updates — this is why gradients can flow through long sequences without vanishing.

07 — Gated Recurrent Unit

GRU — The Lighter Alternative

Introduced by Cho et al. (2014), the GRU simplifies LSTM by merging the cell and hidden state into one, and collapsing to just two gates: a reset gate and an update gate.

GRU Equations
r(t) = σ( W_r · [h(t−1), x(t)] )
z(t) = σ( W_z · [h(t−1), x(t)] )
h̃(t) = tanh( W · [r(t) ⊙ h(t−1), x(t)] )
h(t) = (1−z(t)) ⊙ h(t−1) + z(t) ⊙ h̃(t)
r = reset gate · z = update gate (acts like forget+input combined)
Feature Vanilla RNN LSTM GRU
Gates None 3 (f, i, o) 2 (r, z)

PyTorch Implementation

vanilla_rnn_cell.py
# Manual RNN cell — the pure math
class VanillaRNNCell(nn.Module):
    def __init__(self, input_size, hidden_size, output_size):
        super().__init__()
        self.W_xh = nn.Linear(input_size, hidden_size)
        self.W_hh = nn.Linear(hidden_size, hidden_size, bias=False)
        self.W_hy = nn.Linear(hidden_size, output_size)

    def forward(self, x_seq, h0=None):
        batch_size, seq_len, _ = x_seq.shape
        h = h0 if h0 is not None else torch.zeros(batch_size, self.hidden_size)

        outputs = []
        for t in range(seq_len):
            x_t = x_seq[:, t, :]
            h = torch.tanh(self.W_xh(x_t) + self.W_hh(h))  # core equation
            outputs.append(self.W_hy(h).unsqueeze(1))

        return torch.cat(outputs, dim=1), h
lstm_cell_manual.py
# LSTM cell — all four gates explicit
class LSTMCellManual(nn.Module):
    def __init__(self, input_size, hidden_size):
        super().__init__()
        combined = input_size + hidden_size
        self.W_f = nn.Linear(combined, hidden_size)  # forget
        self.W_i = nn.Linear(combined, hidden_size)  # input
        self.W_C = nn.Linear(combined, hidden_size)  # candidate
        self.W_o = nn.Linear(combined, hidden_size)  # output

    def forward(self, x, h_prev, C_prev):
        combined = torch.cat([h_prev, x], dim=1)

        f = torch.sigmoid(self.W_f(combined))       # forget gate
        i = torch.sigmoid(self.W_i(combined))       # input gate
        C_tilde = torch.tanh(self.W_C(combined))    # candidate
        C = f * C_prev + i * C_tilde                # update cell state
        o = torch.sigmoid(self.W_o(combined))       # output gate
        h = o * torch.tanh(C)                       # new hidden state

        return h, C
sentiment_lstm.py
# High-level model using nn.LSTM
class SentimentLSTM(nn.Module):
    def __init__(self, vocab_size, embed_dim, hidden_size,
                 num_classes, num_layers=2, bidirectional=False):
        super().__init__()
        self.embedding = nn.Embedding(vocab_size, embed_dim, padding_idx=0)
        self.lstm = nn.LSTM(
            input_size=embed_dim, hidden_size=hidden_size,
            num_layers=num_layers, batch_first=True,
            dropout=0.3, bidirectional=bidirectional
        )
        directions = 2 if bidirectional else 1
        self.fc = nn.Linear(hidden_size * directions, num_classes)

    def forward(self, x):
        emb = self.embedding(x)
        _, (hidden, _) = self.lstm(emb)
        last = torch.cat([hidden[-2], hidden[-1]], dim=1) \
               if self.lstm.bidirectional else hidden[-1]
        return self.fc(last)


# Training loop with gradient clipping
def train_epoch(model, loader, optimizer, criterion, clip=1.0):
    model.train()
    for X, y in loader:
        optimizer.zero_grad()
        loss = criterion(model(X), y)
        loss.backward()
        # Critical: clip gradients to prevent explosion
        nn.utils.rnn.clip_grad_norm_(model.parameters(), clip)
        optimizer.step()
09 — Applications

When to Use Which Model

Vanilla RNN

Simple sequential tasks

Best as a teaching tool. Practical for very short sequences only — fails beyond ~20 steps due to vanishing gradients.

Learning Short time-series
LSTM

Long-range dependency tasks

The go-to for NLP, time-series, and speech. Handles hundreds to thousands of time steps reliably.

Sentiment analysis Language model Speech
GRU

Fast, efficient NLP

Competitive with LSTM on most tasks while training ~25% faster. Preferred in resource-constrained settings.

Mobile NLP Fast prototyping
Bi-LSTM

Full-context understanding

Sees both past and future tokens. Ideal when the full sequence is available at inference time.

NER POS tagging QA
Seq2Seq (Encoder–Decoder)

Variable-length to variable-length

The encoder compresses a source sequence into a context vector; the decoder generates the output token by token using teacher forcing during training. Adding Attention gives the decoder direct access to all encoder hidden states — the architectural ancestor of Transformers.

Machine translation Summarization Chatbots Code generation
10 — Training Tips

Key Training Practices

  • 1
    Gradient clipping — Always clip gradients (norm=1.0–5.0). Without it, a single bad batch can send weights to infinity. Use clip_grad_norm_(model.parameters(), 1.0) before each optimizer step.
  • 2
    Dropout placement — Apply dropout between LSTM layers, not inside the recurrence loop. PyTorch handles this automatically when dropout > 0 and num_layers > 1.
  • 3
    Packed sequences — For variable-length batches, use pack_padded_sequence() and pad_packed_sequence() so the LSTM doesn't waste computation on padding tokens.
  • 4
    Teacher forcing — In seq2seq models, feed the real target (not prediction) to the decoder with probability p ≈ 0.5 during training. Curriculum learning schedules that anneal p from 1 → 0 work well for long generation tasks.
  • 5
    LR scheduling — Use ReduceLROnPlateau or cosine annealing. RNNs are sensitive to learning rate — a schedule that halves LR on validation plateau is often sufficient.
  • 6
    RNNs vs Transformers — For most NLP tasks, Transformers now dominate. RNNs remain the best choice for streaming inference, time-series, embedded/mobile, and RL environments where step-by-step processing is natural and the full sequence isn't available upfront.
